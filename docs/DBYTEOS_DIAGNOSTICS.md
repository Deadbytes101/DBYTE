# DByteOS Diagnostics

DByteOS System Diagnostics provide commands for checking system readiness and identifying issues across subsystems.

## Commands

- `doctor`: Top-level health check covering all subsystems.
- `diagnose [subsystem]`: Drill-down check for specific subsystems (profile, config, security, logs, manual, package).
- `check-system`: Quick readiness check before opening mutable state.

## Subsystems

- Profile
- Config
- Security
- Logs
- Manual
- Package
