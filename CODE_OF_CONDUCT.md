# Code of Conduct

DBYTE is a technical project for language tooling, byte-level automation, DByteOS userland experiments, and Kernel Lab research.

## Expected Behavior

- Keep discussion focused on the code, design, tests, docs, and reproducible behavior.
- Be direct without being abusive.
- Respect maintainers, contributors, and users.
- Assume experimental areas can break and document failures clearly.
- Keep security-sensitive reports in the security reporting path.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or targeted abuse.
- Publishing private information without permission.
- Spamming issues, pull requests, discussions, or support channels.
- Derailing technical work with unrelated fights.
- Encouraging unsafe execution of Kernel Lab or VM workflows outside documented boundaries.

## Enforcement

Maintainers may edit, hide, close, lock, or remove comments, issues, pull requests, and discussions that disrupt the project.

Repeated or severe violations may result in a ban from project spaces.

## Reporting

Report conduct issues through GitHub moderation tools or by contacting the maintainer through the repository profile.
